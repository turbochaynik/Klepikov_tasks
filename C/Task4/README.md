echo:
Описание программы
Программа выполняет следующие шаги:

Обработка флагов:

Если первый аргумент начинается с -, программа интерпретирует его как флаг.
Флаг -n означает, что не нужно добавлять новую строку (\n) в конце вывода.
Флаг -e активирует интерпретацию экранированных символов, а флаг -E деактивирует эту возможность.
Обработка строк:

Программа последовательно добавляет каждое слово (аргумент командной строки) в буфер.
При этом она учитывает флаг -e, интерпретируя экранированные символы (например, \n, \t, и другие).
Память:

Программа динамически увеличивает размер буфера, если в нем недостаточно места для добавляемых данных.
Вывод:

После обработки всех аргументов программа выводит строку на стандартный вывод. Если не был установлен флаг -n, в конец строки добавляется символ новой строки (\n).
Основные функции
ensure_capacity:

Эта функция проверяет, хватает ли памяти в буфере для добавления новых символов. Если памяти недостаточно, она увеличивает размер буфера в два раза с помощью realloc.
addword:

Эта функция добавляет слово (строку) в буфер. Если активирован флаг -e, она интерпретирует экранированные символы в строке. Например, она заменяет \n на символ новой строки, \t — на табуляцию, а также поддерживает восьмеричную и шестнадцатеричную запись символов.
main:

Функция main обрабатывает аргументы командной строки, активируя нужные флаги и добавляя слова в буфер. В конце она выводит результат на стандартный вывод и освобождает память.


pwd:
Описание программы
Программа выполняет следующие действия:

Получение текущей рабочей директории:

Программа использует системный вызов getcwd, чтобы получить полный путь к текущей рабочей директории и сохранить его в буфере str.
Обработка ошибок:

Если при вызове getcwd возникает ошибка, устанавливается глобальная переменная errno. Программа проверяет значение errno и, если ошибка произошла, выводит соответствующий код ошибки.
Вывод текущей директории:

Если вызов getcwd завершился успешно, программа выводит путь к текущей рабочей директории.


ls:
Описание программы
Программа выполняет следующие задачи:

Обработка флагов командной строки:

Программа позволяет пользователю задать флаги для управления выводом информации:
-R: Рекурсивный режим. При этом программа будет искать файлы и в подкаталогах.
-l: Длинный формат. Выводится подробная информация о файле (права, владелец, размер и т.д.).
-g: Включает отображение имени группы, вместо UID владельца.
Вывод содержимого директории:

Программа выводит файлы и каталоги в указанной директории. При этом могут быть использованы дополнительные параметры:
Для флага -l: Выводится информация о файле (права доступа, владелец, группа, размер).
Для флага -g: Вместо UID владельца выводится имя группы.
Рекурсивный обход:

При включении флага -R программа будет рекурсивно обходить все подкаталоги, выводя их содержимое. Исключение составляют каталоги с именем .git, которые не будут обрабатываться.
Основные функции
parse_flags:

Эта функция анализирует строку флагов и устанавливает соответствующие флаги для режима работы программы:
-R — для рекурсивного обхода.
-l — для длинного формата вывода.
-g — для отображения группы вместо UID.
print_file_info:

Функция для вывода информации о файле:
Если флаг -l активирован, выводится подробная информация (права доступа, владелец, группа, размер).
Если флаг -g активирован, выводится имя группы вместо UID владельца.
Для символических ссылок выводится путь к целевому файлу.
process_directory:

Функция для обработки содержимого директории:
Если флаг -R активирован, программа рекурсивно обрабатывает все подкаталоги.
Для каждого файла или каталога вызывает функцию print_file_info, чтобы вывести информацию.
main:

Основная функция программы:
Извлекает текущую директорию с помощью getcwd.
Обрабатывает флаги командной строки.
Вызывает process_directory, чтобы вывести содержимое директории.

cp:
Описание программы
Программа выполняет следующие шаги:

Проверка аргументов: Программа принимает два аргумента — исходный файл или каталог и целевой файл или каталог. Если количество аргументов отличается от двух, программа выведет сообщение об ошибке и завершит работу.

Проверка одинаковости файлов: Программа проверяет, что исходный и целевой файлы не совпадают. Если они одинаковы, программа выведет ошибку и завершится.

Проверка существования целевого файла: Если целевой файл существует, программа получает его режим (права доступа).

Копирование файлов:

Если исходный объект — файл: Программа открывает исходный файл для чтения и целевой файл для записи. Копирование происходит поблочно (по 256 байт). По завершении копирования программа устанавливает такие же права доступа для целевого файла, как у исходного.
Если исходный объект — каталог: Программа открывает исходный каталог и создает новый каталог в целевом месте. Затем рекурсивно копирует все содержимое исходного каталога в целевой каталог, обходя все вложенные файлы и каталоги.
Рекурсивное копирование для каталогов: Если исходный объект — каталог, программа использует рекурсию для копирования содержимого каталога, создавая аналогичные каталоги и копируя файлы внутри.


cmd:
Описание программы
Программа выполняет следующие действия:

Проверка аргументов: Программа ожидает два аргумента — пути к двум файлам для сравнения. Если количество аргументов отличается от двух, программа выведет сообщение об ошибке и завершит работу.

Открытие файлов: Программа пытается открыть два файла в режиме чтения. Если не удается открыть хотя бы один из файлов, программа выведет ошибку с причиной (например, если файл не существует или отсутствуют права доступа) и завершится.

Сравнение файлов: Программа сравнивает файлы побайтово:

Если файлы идентичны, программа завершится без вывода.
Если файлы различаются, программа выведет сообщение с номером байта и строки, где произошла ошибка.
Если один из файлов заканчивается раньше другого, программа выведет сообщение о достижении конца файла.
Отслеживание состояния: Программа отслеживает текущий номер байта и строки для точности сообщения о различиях. Если различие находится в конце строки, она корректно учитывает изменение номера строки.
